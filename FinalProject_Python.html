About the Project - Python

The objective of this project is to forecast customer churnby utilizing a dataset that contains more than 20 features that have the potential to impact retention. For Python analysis, two models are chosen: Support Vector Machine (SVM) and Logistic Regression.

Binary classification tasks are well-suited to Logistic Regression, which provides high interpretability, enabling businesses to comprehend the extent to which individual features contribute to attrition risk. By contrast, SVM is a more intricate algorithm that may not offer the same level of transparency, but it frequently provides superior predictive performance, particularly in situations involving non-linear relationships and imbalanced data.

The project offers a balanced approach by integrating both an interpretable model and a more powerful, less interpretable one. This approach provides actionable insights through Logistic Regression and potentially higher accuracy and recall through SVM. This dual-model strategy provides businesses with adaptable options that are contingent upon their priorities, whether it be the optimization of predictive effectiveness or the comprehension of consumer behavior.

The primary objective is to enhance the model's capacity to accurately identify consumers who are at risk of churn. In a business setting, false positives (predicting churn when the customer would remain) are generally less expensive, as proactive retention efforts may still strengthen customer loyalty. Nevertheless, false negatives (the failure to identify a consumer who will churn) can lead to missed opportunities for intervention and lost revenue. Consequently, the model is assessed with a focus on recall for the attrition class to guarantee that retention strategies are effectively identified among at-risk customers.

One-Hot Encoding Features

One-hot encoding turns multi-value features into binary features. For example, ['MultipleLines'] has 3 possible values: 'Yes,' 'No,' and 'No Phone Service.' To make these features binary, each possible value must become its own binary column such that column ['MultipleLines_Yes'] is binary to whether the customer said yes (1) to having multiple lines or no phone service/no (0).

To prevent perfect multicollinearity, which refers to the highly related dummy variables, one of each of the dummy variables will be removed. These features are highly related because if InternetService_FiberOptic and InternetService_No are both 0 values, then InternetService_DSL is highly predictable and causes issues for the logistic regression model and the SVM. These dummy features that are going to be omitted will coincide with those from the R Project, and they will only be omitted from the model, not dropped from the dataset.

Additionally, there is high multicollinearity with PhoneService and MultipleLines_No Phone Service. That being said, PhoneService will be omitted for easier implementation in both this Python project and the R project.

Troubleshooting Data

After initially running this model, there were issues with some features being multicollinear. In a model where there is no multicollinearity, values in the coefficient table would all contain metrics; however, issues when there are missing values in the table indicate the presence of multicollinearity for those features. To determine which of these features is causing the lack of metrics in the table, a variance inflation factor (VIF) can be run to score each dataset feature. A feature with a high VIF value (VIF > 10) indicates a large similarity/multicollinearity with another feature.

After running a variance inflation factor function, PhoneService scored a 14, while the second highest was MonthlyCharges at 4.8. This shows the large gap between VIF scores and how high PhoneService scored on the VIF. ChatGPT then provided code that could determine which feature PhoneService was highly correlated to, and after running the code, it was determined to be the MultipleLines_No Phone Service. This would make sense, as the one-hot encoded MultipleLines feature essentially encompassed everything the PhoneService feature had to offer. Therefore, the best way of action is to omit the PhoneService feature from the models entirely.

Support Vector Machine (SVM)

The SVM classification model works by finding the optimal hyperplane that separates the binary target values. When new data is then input into the model, it is classified based on if it falls above the hyperplane or below the hyperplane.

The heatmap visualization reveals that the SVM model correctly identified 218 customers who churned, while incorrectly labeling 112 customers as churners. This level of false positives is acceptable in the business context, as previously discussedit's generally preferable to mistakenly flag a customer as likely to churn than to miss one who actually does. On the other hand, the model correctly classified 924 customers as non-churners but failed to identify 155 actual churners, misclassifying them as non-churners. This group represents a critical loss, as these customers were not targeted for retention efforts and ultimately left the service.

Although this is still a large number of lost customers, in terms of model strength, it shows that the SVM failed to identify around 6 less churning customers than the Logistic Regression Model.

Conclusion

The Support Vector Machine (SVM) model outperformed Logistic Regression in terms of predictive performance, although only marginally. Despite its slightly inferior performance, logistic regression provided valuable insights into how various characteristics influence churn, delivering interpretability that SVM lacked.

Given this trade-off, Logistic Regression may be a more realistic option for business applications where understanding churn reasons is just as critical as forecast accuracy. Its transparency allows for data-driven decision-making and tailored retention measures, making it a great contender despite a slight performance gap compared to SVM.
